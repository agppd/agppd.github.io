## 一、进度

> 进度（Advancements）是一种用来逐步引导新手玩家融入Minecraft，并给玩家提供挑战的系统。它类似于原主机版和基岩版中更简单的成就系统。——Wiki

但是，进度在数据包开发中，不仅是用来引导玩家的，而一般是用来检测玩家操作的。

### 定义格式

数据包中，进度由JSON文件定义，位于`<数据包名称>/data/<命名空间>/advancement`目录下。

#### 文件结构
```json
{//根标签
    "criteria":{//一系列准则（见下文）

    },
    "display":{//显示信息（见下文）

    },
    "parent":"",//此进度的父进度
    "requirements":[
        /*此进度如何达成，包含若干包含一系列准则的子列表，当每个子列表中都至少有一个准则达成时，该进度达成*/
        [
            "准则名称"
        ]
    ],
    "rewards":{//进度达成后的奖励（并非）
        "experience":114514,//玩家获得的经验值（默认为0）
        "function":"function_name",//达成后执行的函数，不支持标签。
        "loot":[//完成后给与玩家的战利品表
            "loot_table_name"
        ],
        "recipes":[//完成后玩家解锁的配方
            "recipe_name"
        ]
    },
    "sends_telemetry_event":false//是否发送遥测数据，命名空间不在minecraft无效，默认为false
}
```

#### 准则

```json
{
    "<准则名称>":{
        "trigger":"***",//触发器的ID
        "conditions":{//触发后达成准则需要的条件（其余项为各触发器特有的）
            "player":[//一个或一些谓词，触发的玩家需要满足的条件

            ]
        }
    }
}
```

在游戏中定义的准则触发器以及每项细节详见Wiki。

#### 显示

##### 定义格式

```json
{
    "display":{
        "announce_to_chat":true,//是否在聊天窗口提示
        "background":"namespace:bkg",//作为进度背景平铺图片的纹理。
        "description":"描述信息",
        "frame":"task",//图标边框
        "hidden":false,//是否为隐藏进度
        "icon":{
            //物品堆叠，表示图标
        },
        "title":"进度显示名称",
        "show_toast":true//是否右上角显示提示
    }
}

```
##### 选项卡

创建带有显示数据NBT复合标签/JSON对象display的根进度（不含字符串parent的进度）后将自动在进度菜单边缘创建一个选项卡。当游戏加载数据包后，若根进度所在的进度树中有任意进度被某玩家获取，则该根进度以及其选项卡都会显示给该玩家，且根进度和选项卡的图标一致。

根进度的背景图像不会影响选项卡的创建。若根进度未正确从字符串background加载图片，则背景默认显示为丢失纹理。

如果一个根进度成功创建了一个选项卡，它的子进度将会展示在该选项卡中（前提是该子进度拥有显示数据 字符串display）。

### 应用

当一个进度没有`display`标签时，它不会被显示，我们可以在`rewards`中设置执行函数，然后将此进度的完成移除来实现触发器功能。

## 二、谓词

>谓词（Predicate）也即战利品表谓词（Loot Predicate），用于判定某些对象或参数是否满足某种特性或条件。在数据包中，谓词可通过位于data/<命名空间>/predicate/下的JSON文件来定义。 ——Wiki

谓词可以通俗的理解为一个布尔函数，它拥有可被检测的通过/失败的返回值。谓词一般在以下情景被调用：

1. 目标选择器中的`predicate=`。
2. `execuate` 的条件分支中的 `predicate`。
3. 其他谓词调用。
4. 进度和战利品表调用。

数据格式详见Wiki。

## 三、战利品表

战利品表（Loot Table）用于决定游戏在何种情况下生成何种物品。

### 调用

战利品表决定大部分情况下的掉落物，容器内容物，钓鱼，猪灵以物易物等。

玩家可以在容器中添加NBT标签来使其第一次打开时会通过战利品表生成内容物。

也可以用指令调用战利品表。

### 定义

详见Wiki。

大概是有多个随机池，每个随机池会抽取若干次，有幸运值决定（幸运buff的唯一用处）。

每个随机池及每个随机池中的抽取项都可以用物品修饰器修饰，用于改变物品数量和nbt。

同时每个随机池可以根据谓词判断是否应用。
