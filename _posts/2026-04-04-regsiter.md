## 一、准备工作：先备份当前注册表

**在进行任何修复操作之前，务必备份当前注册表。** 备份操作步骤如下：

1. 按下 `Win + R` 键打开“运行”对话框
2. 输入 `regedit` 并按回车，以管理员身份打开注册表编辑器
3. 在注册表编辑器窗口中，点击“文件” → “导出”
4. 选择保存位置，输入文件名，在“导出范围”中选择“全部”，点击“保存”

> **提示**：备份整个计算机的注册表文件耗时较长，备份过程中可能会失去响应，请勿强行结束程序。


## 二、常用修复方法

### 方法一：使用系统文件检查器（SFC）修复

SFC 是 Windows 内置工具，可以扫描并修复损坏的系统文件，包括注册表项。

**操作步骤**：

1. 在搜索栏中输入 `cmd` 或 `命令提示符`
2. 右键点击“命令提示符”，选择 **“以管理员身份运行”**
3. 在命令提示符窗口中输入以下命令并回车：
   ```
   sfc /scannow
   ```
4. 等待扫描完成（通常需要 30 分钟左右），系统会自动修复检测到的问题
5. 扫描完成后重启计算机

> **注意**：如果运行后提示“Windows Resource Protection found corrupt files but was unable to fix some of them”，请重启计算机后再次尝试。

### 方法二：使用 DISM 工具修复

DISM（部署映像服务和管理）是另一个 Windows 内置工具，用于修复 Windows 映像和注册表，通常与 SFC 配合使用效果更佳。

**操作步骤**：

1. 以管理员身份打开命令提示符或 Windows PowerShell
2. 先执行扫描检查命令：
   ```
   DISM /Online /Cleanup-Image /ScanHealth
   ```
3. 再执行修复命令：
   ```
   DISM /Online /Cleanup-Image /RestoreHealth
   ```
4. 等待进度完成（通常需要 5–20 分钟）
5. 执行完成后立即输入 `sfc /scannow` 再次扫描
6. 若提示“已修复某些文件”，重启计算机使更改生效

### 方法三：使用 RegBack 文件夹恢复注册表

Windows 系统会在每次成功启动后，自动将注册表配置单元备份至 `C:\Windows\System32\config\RegBack` 目录。此方法无需图形界面即可操作，适合严重故障场景。

**操作步骤（系统仍可启动时）**：

1. 以管理员身份打开命令提示符
2. 切换至 config 目录：
   ```
   cd /d C:\Windows\System32\config
   ```
3. 依次执行以下命令（每条执行后需确认无报错）：
   ```
   ren SYSTEM SYSTEM.bak
   copy ..\RegBack\SYSTEM SYSTEM
   
   ren SOFTWARE SOFTWARE.bak
   copy ..\RegBack\SOFTWARE SOFTWARE
   ```
4. 对 DEFAULT、SECURITY、SAM 重复相同操作
5. 完成后重启计算机

**操作步骤（系统无法启动时）**：

1. 使用 Windows 安装介质（启动 U 盘）启动电脑
2. 选择“修复计算机” → “疑难解答” → “命令提示符”
3. 在命令提示符中确认 RegBack 目录存在：
   ```
   dir C:\Windows\System32\config\RegBack
   ```
4. 执行复制命令覆盖原注册表文件，完成后退出的 exit，重启计算机

> **注意**：部分精简版系统可能缺失备份功能，恢复的注册表可能非最新版本。

### 方法四：使用系统还原恢复

如果系统开启了系统还原功能并创建过还原点，可以通过系统还原来恢复注册表。

**操作步骤**：

1. 按下 `Win + R` 键，输入 `rstrui` 并回车，启动系统还原向导
2. 点击“下一步”，在列表中选择问题发生前创建的还原点
3. 勾选“在还原前创建还原点（推荐）”
4. 点击“下一步”后确认执行
5. 系统将自动重启并开始还原，过程中不可强制关机或中断操作

### 方法五：使用“最后一次正确的配置”

当注册表错误导致系统无法正常启动时，可以尝试使用此方法。

**操作步骤**：

1. 重启计算机，在 BIOS 自检结束后、Windows 徽标出现前连续按 `F8` 键（部分新设备需按 `Shift + F8`）
2. 在高级启动选项菜单中，使用方向键选中 **“最后一次正确的配置（高级）”**
3. 按回车确认，系统将尝试以该配置启动

### 方法六：手动导入已备份的 .reg 文件

如果之前已导出并保存了 .reg 格式的注册表备份文件，可以直接导入恢复。

**操作步骤**：

1. 双击需要导入的 .reg 文件
2. 在弹出的确认对话框中点击“是”
3. 导入成功后系统会提示“已成功添加”，点击“确定”完成

也可以通过注册表编辑器导入：打开注册表编辑器 → 点击“文件” → “导入” → 选择 .reg 文件 → 点击“打开”。

### 方法七：手动编辑注册表（高级用户）

对于有一定计算机知识的用户，可以尝试手动编辑注册表来修复问题，但此操作风险较高，需要非常谨慎。

**操作步骤**：

1. 打开注册表编辑器（regedit）
2. 导航到损坏的注册表项，尝试删除或修改错误的键值
3. 修改完成后重启计算机

> **重要提醒**：手动编辑注册表之前，务必先备份注册表。错误的编辑可能导致系统无法启动。

### 方法八：重置 Windows（最后手段）

如果以上所有方法都无法修复注册表，且注册表损坏严重，可以考虑重置 Windows。

**操作步骤**：

1. 打开“设置”（`Win + I`）
2. 选择“更新和安全” → “恢复”
3. 点击“重置此电脑”
4. 根据提示选择“保留我的文件”或“删除所有内容”
5. 按照屏幕指示完成重置
