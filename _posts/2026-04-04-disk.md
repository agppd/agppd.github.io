## 一、机械硬盘（HDD）的物理结构

### 1. 基本组成
- 盘片：圆形铝合金或玻璃基片，表面涂有磁性介质。多张盘片叠装在同一主轴上。
- 磁头：每个盘面有一个独立的读写磁头，悬浮在盘片上方约5–10纳米处。
- 主轴电机：带动盘片以恒定角速度旋转（常见转速：5400、7200、10000、15000 RPM）。

——by DS

### 2. 磁道（Track）
盘片从圆心向外被划分成多个同心圆，每个圆称为一个磁道。最外圈磁道最长，存储数据量最大。

### 3. 扇区（Sector）
每条磁道被分割成若干弧段，每个弧段是一个扇区。扇区是磁盘**物理读写的最小单位**。传统扇区大小为512字节，现代大容量硬盘采用4096字节（4K扇区）。扇区之间留有间隙用于同步和分隔。

### 4. 柱面（Cylinder）
所有盘片上相同半径的磁道在三维空间上形成一个虚拟圆柱，称为柱面。早期CHS寻址使用柱面号、磁头号、扇区号唯一确定一个扇区。

### 5. 寻址方式演进
- CHS（Cylinder-Head-Sector）：直接使用物理几何参数，受限于硬盘容量。
- LBA（Logical Block Addressing）：将整个磁盘抽象为从0开始的连续逻辑块，由硬盘控制器内部映射到物理位置。现代操作系统均使用LBA。

### 6. 分区位记录（ZBR）
将盘片划分为多个区域，外圈区域的每条磁道包含更多扇区，内圈较少。因此外圈的数据传输速率高于内圈。

### 7. 坏道
- 逻辑坏道：数据校验错误，可通过重写或低级格式化修复。
- 物理坏道：盘片磁性介质损伤，不可修复，由固件重映射到备用区。

## 二、固态硬盘（SSD）

### 1. 存储介质
NAND闪存，由浮栅晶体管组成。根据每个单元存储的比特数分类：
- SLC（1 bit/cell）：速度最快、寿命最长（约10万次擦写），成本高。
- MLC（2 bit/cell）：约3000–10000次擦写。
- TLC（3 bit/cell）：主流消费级，约1000–3000次擦写。
- QLC（4 bit/cell）：容量大、成本低，寿命约500–1000次擦写。

### 2. 组织层次
- 页（Page）：SSD读写的最小单位，通常为4KB、8KB或16KB。不能单独覆盖写，必须先擦除。
- 块（Block）：由数百个页组成，是SSD擦除的最小单位。

### 3. 闪存转换层（FTL）
SSD主控中的固件，负责LBA到物理页/块的地址映射。主要功能：
- 地址映射：维护映射表。
- 垃圾回收（GC）：将有效页迁移到新块，擦除无效页多的旧块。
- 磨损均衡：使所有物理块擦除次数接近，防止部分块过早损坏。

### 4. TRIM指令
操作系统删除文件后，通过TRIM指令告知SSD哪些LBA已无效。SSD可将对应物理页标记为可回收，减少写入放大。

### 5. 接口与协议
- SATA + AHCI：最大理论带宽6Gbps（约560MB/s实际），协议为机械硬盘优化。
- NVMe over PCIe：低延迟、多队列，专为SSD设计。PCIe 3.0 x4约3.5GB/s，PCIe 4.0 x4约7GB/s，PCIe 5.0 x4约14GB/s。

### 6. 寿命指标
- TBW（Total Bytes Written）：保修期内保证的总写入字节数。
- DWPD（Drive Writes Per Day）：每日全盘写入次数，企业级常用。

### 7. 4K对齐
文件系统的簇（通常4KB）若对齐到SSD页边界，可避免一次写入跨两个物理页，减少写入放大。

## 三、分区

### 1. 分区表类型

**MBR（主引导记录）**
- 最大支持2TB磁盘（512字节扇区）。
- 最多4个主分区，或3个主分区+1个扩展分区（内含多个逻辑分区）。
- 引导方式：BIOS/Legacy。

**GPT（GUID分区表）**
- 最大支持18EB。
- 理论上最多128个分区（Windows限制）。
- 磁盘末尾有备份分区表，带CRC32校验。
- 引导方式：UEFI。

### 2. 分区工具
- Windows：磁盘管理、diskpart。
- Linux：fdisk、gdisk、parted。

## 四、文件系统

### 1. 作用
- 组织文件和目录结构。
- 记录每个文件占用的存储位置（簇/块）。
- 管理空闲空间。
- 维护文件属性（权限、时间戳等）。

### 2. 常见文件系统
- NTFS：Windows主流，支持大文件、权限、压缩、加密、日志。
- FAT32：跨平台兼容，单文件≤4GB，分区≤2TB。
- exFAT：跨平台，无4GB限制，无日志，适合U盘/移动硬盘。
- ext4：Linux主流，成熟稳定，支持日志、延迟分配。
- APFS：macOS原生，针对SSD优化，支持快照、克隆。
- XFS：Linux下适合大文件和高并发。

### 3. 核心内部结构（以ext4为例）
- 超级块：存储文件系统整体信息（块总数、空闲块数、inode表位置等）。
- 块组：将分区划分为多个块组，每组包含：
  - 块位图：标记数据块使用状态。
  - inode位图：标记inode使用状态。
  - inode表：存放每个文件/目录的inode。
  - 数据块：实际文件内容。
- inode：每个文件对应一个inode，存储文件大小、权限、时间戳、指向数据块的指针。文件名存储在目录项中，目录项是“文件名→inode号”的映射。

### 4. 簇（块）
文件系统分配空间的最小单位，由连续的若干个扇区组成。常见大小为4KB。小文件会占用整个簇，可能造成内部碎片。

### 5. 日志（Journaling）
在修改元数据前，先将操作记录到日志区。系统异常断电后，回放日志即可恢复一致性，无需全盘扫描。NTFS、ext3/4、XFS均支持。

### 6. 写时复制（CoW）
修改文件时写入新位置，最后原子更新指针。便于实现快照和避免数据损坏。btrfs、ZFS、APFS采用。

## 五、HDD与SSD对比

| 特性 | HDD | SSD |
|------|-----|-----|
| 读写原理 | 磁头寻道+盘片旋转 | 电子寻址，无机械运动 |
| 访问延迟 | 5–12毫秒 | 0.02–0.1毫秒 |
| 连续读写 | 80–260 MB/s | SATA: 550 MB/s; NVMe: 3–14 GB/s |
| 随机IOPS | <200 | 数十万至百万 |
| 抗震 | 差 | 好 |
| 噪音 | 有 | 无 |
| 功耗 | 5–10W | 2–5W |
| 每GB成本 | 低 | 较高 |
| 写入寿命 | 无理论上限 | 有限（日常使用足够） |
