## 二、

### 什么是数据包？

> 数据包（Data Pack）为玩家自定义Minecraft的游戏内容提供了更多新方法，包括但不限于配置进度、配方、战利品表、魔咒、伤害类型、生物变种和世界生成等。对数据包的修改并不等于修改了包含相同逻辑部分的游戏代码。 ——Wiki

### 开发工具

一般情况下使用VScode开发数据包，需要插件：
1. Datapack Helper Plus by Spyglass 必装插件，提供语法高亮，自动补全和纠错。
2. Datapack Icons 可选，提供图标。
3. NBT Viewer 可选，查看NBT。

### 文件结构

一般数据包的结构为：

```txt
<数据包名称>
  Lpack.mcmeta 数据包元数据
  Lpack.png 图标
  Ldata
   L<命名空间>
    L...
```

#### 元数据

数据包需要指定元数据才能被游戏识别为一个数据包。数据包元数据位于数据包根目录，其名称为pack.mcmeta，使用JSON格式，例如：
```json
{
    "pack":{
        "description": "示例数据包",//数据包描述
        "pack_format": 71,//已弃用，为兼容性而保留。数据包的基础版本。
        "supported_formats": [48, 81],//已弃用，为兼容性而保留。此数据包支持的数据包版本范围。
        "min_format": 48,//最低版本
        "max_format": [88, 0]//最高版本
    }
}
```

### 添加数据包

在创建世界前，玩家可以在创建新的世界屏幕的“更多”选项卡中配置数据包。与配置资源包的界面类似，玩家可以自由选择数据包的加载顺序和启用情况，并可以搜索数据包。

存档被创建后，玩家只能直接在存档的<存档根目录>/datapacks目录下添加数据包。
