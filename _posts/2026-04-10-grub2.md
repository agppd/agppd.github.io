## 一、GRUB2 的核心配置文件

### 1.1 配置文件的“三层结构”

GRUB2 的配置体系由三层文件构成，这是理解其运作的关键：

**第一层：主配置文件 `/boot/grub/grub.cfg`**
这是 GRUB2 启动时实际读取的配置文件，包含了所有菜单项的完整定义。**⚠️ 此文件禁止直接编辑！** 它是由 `grub-mkconfig` 命令根据下层源文件自动生成的，任何手动修改都会在下次更新配置时被覆盖。

**第二层：用户设置文件 `/etc/default/grub`**
这是用户进行全局配置的核心文件，控制着菜单外观、超时时间、默认启动项和内核参数等。修改这个文件不会直接生效，必须运行更新命令重新生成 `grub.cfg`。

**第三层：脚本目录 `/etc/grub.d/`**
该目录下的脚本文件按数字前缀顺序执行，共同构建最终的菜单内容：
- `00_header`：加载 `/etc/default/grub` 中的全局设置
- `10_linux`：搜索 Linux 内核并生成对应菜单项
- `30_os-prober`：探测其他操作系统（如 Windows）
- `40_custom`：用户自定义菜单项的模板文件

### 1.2 `/etc/default/grub` 常用配置项详解

```bash
# 默认启动项序号（从0开始计数）
GRUB_DEFAULT=0

# 菜单超时时间（秒），设为-1表示无限等待用户选择
GRUB_TIMEOUT=5

# 菜单显示风格：menu（显示菜单）、hidden（隐藏菜单，可按Esc唤出）、countdown（显示倒计时）
GRUB_TIMEOUT_STYLE=menu

# 图形模式分辨率设置
GRUB_GFXMODE="1920x1080,auto"

# 传递给 Linux 内核的额外参数
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"

# 是否禁用 OS 探测（设为false以启用其他操作系统探测）
GRUB_DISABLE_OS_PROBER=false

# 主题路径
GRUB_THEME="/boot/grub/themes/mytheme/theme.txt"

# 背景图片路径（单独设置背景图时使用）
GRUB_BACKGROUND="/boot/grub/background.png"
```

### 1.3 更新配置使其生效

修改配置文件后，必须运行以下命令重新生成 `grub.cfg`：

```bash
# Debian/Ubuntu 系列
sudo update-grub

# RHEL/CentOS/Fedora 系列
sudo grub2-mkconfig -o /boot/grub2/grub.cfg
```

`update-grub` 实际上是 `grub-mkconfig -o /boot/grub/grub.cfg` 的封装命令。


## 二、常用命令行工具

### 2.1 grub-install：安装与修复 GRUB

`grub-install` 用于将 GRUB2 安装到磁盘的主引导记录（MBR）或 EFI 分区。

```bash
# BIOS/MBR 模式：安装到整个硬盘
sudo grub-install /dev/sda

# UEFI 模式：安装到 EFI 分区
sudo grub-install --target=x86_64-efi --efi-directory=/boot/efi --bootloader-id=GRUB

# 安装到可移动设备（如 USB 启动盘）
sudo grub-install --target=x86_64-efi --efi-directory=/mnt/usb --boot-directory=/mnt/usb/boot --removable
```

对于 BIOS 系统，建议始终使用 `--target=i386-pc` 选项来明确指定安装目标。

### 2.2 grub-mkconfig：生成配置文件

这是 `update-grub` 背后的实际工具，可根据 `/etc/default/grub` 和 `/etc/grub.d/` 下的脚本生成配置文件。

```bash
# 生成配置文件并输出到指定位置
sudo grub-mkconfig -o /boot/grub/grub.cfg
```

### 2.3 grub2-mkpasswd-pbkdf2：生成加密密码

用于为 GRUB2 生成加密的密码哈希，在设置密码保护时需要用到。

```bash
sudo grub2-mkpasswd-pbkdf2
# 输入密码后，会输出类似如下格式的哈希字符串
# grub.pbkdf2.sha512.10000.xxxxxxxx...
```

### 2.4 grub2-set-default：设置默认启动项

配合 `GRUB_DEFAULT=saved` 使用，可以动态设置下次启动的默认菜单项。

```bash
# 查看所有菜单项
awk -F\' '$1=="menuentry " {print $2}' /boot/grub/grub.cfg

# 将第3个菜单项设为默认（序号从0开始）
sudo grub2-set-default 2
```

### 2.5 grub2-setpassword：快速设置密码

一键设置 GRUB2 编辑权限密码，会自动生成 `/boot/grub2/user.cfg` 文件。

```bash
sudo grub2-setpassword
# 按提示输入两次密码即可
```


## 三、多系统引导配置

### 3.1 自动探测其他操作系统

GRUB2 通过 `os-prober` 工具自动探测硬盘上的其他操作系统（如 Windows、其他 Linux 发行版）。

**步骤一：安装 os-prober**

```bash
# Ubuntu/Debian
sudo apt install os-prober

# Fedora/RHEL
sudo dnf install os-prober
```

**步骤二：启用 OS 探测**

编辑 `/etc/default/grub`，确保包含以下配置：

```bash
GRUB_DISABLE_OS_PROBER=false
```

**步骤三：更新配置**

```bash
sudo update-grub
```

如果更新后仍看不到 Windows 启动项，可能需要先挂载 EFI 分区（UEFI 系统）：

```bash
# 查看 EFI 分区（通常是 /dev/sda1 或 /dev/nvme0n1p1）
sudo fdisk -l
# 临时挂载
sudo mount /dev/nvme0n1p1 /boot/efi
# 重新更新配置
sudo update-grub
```

### 3.2 手动添加自定义菜单项

当自动探测不生效时，可以通过编辑 `/etc/grub.d/40_custom` 手动添加启动项。

**添加 Windows 启动项示例（UEFI）：**

```bash
#!/bin/sh
exec tail -n +3 $0
# This file provides an easy way to add custom menu entries.

menuentry 'Windows 11' {
    insmod part_gpt
    insmod fat
    insmod chain
    search --fs-uuid --no-floppy --set=root XXXX-XXXX
    chainloader (${root})/EFI/Microsoft/Boot/bootmgfw.efi
}
```

其中 `XXXX-XXXX` 需替换为 Windows EFI 分区的 UUID，可通过 `sudo blkid` 命令查看。

**添加其他 Linux 发行版示例：**

```bash
menuentry 'Ubuntu 22.04' {
    insmod ext2
    set root='hd0,gpt5'
    linux /boot/vmlinuz root=/dev/sda5 ro quiet splash
    initrd /boot/initrd.img
}
```

修改后运行 `sudo update-grub` 即可生效。

### 3.3 设置默认启动系统

有多种方式设置默认启动项：

```bash
# 方式一：在 /etc/default/grub 中使用菜单标题
GRUB_DEFAULT="Ubuntu"

# 方式二：使用序号（0 为第一个菜单项）
GRUB_DEFAULT=2

# 方式三：使用“记住上次选择”模式
GRUB_DEFAULT=saved
GRUB_SAVEDEFAULT=true
```


## 四、GRUB2 主题美化

### 4.1 主题文件结构

GRUB2 主题是一组文件，包括图像、颜色方案和配置文件 `theme.txt`，用于自定义启动菜单的外观。一个标准的主题目录结构如下：

```
/boot/grub/themes/mytheme/
├── theme.txt          # 主题配置文件（必需）
├── background.png     # 背景图片
├── terminal-*.pf2     # 字体文件
└── icons/             # 图标目录（可选）
```

### 4.2 获取与安装主题

可以从以下网站下载现成的主题：

- **GNOME-Look**：[https://www.gnome-look.org/browse?cat=109](https://www.gnome-look.org/browse?cat=109)
- **GitHub 主题仓库**：[https://github.com/RomjanHossain/Grub-Themes](https://github.com/RomjanHossain/Grub-Themes)

**安装步骤：**

```bash
# 1. 创建主题目录
sudo mkdir -p /boot/grub/themes/mytheme

# 2. 解压下载的主题文件到该目录
sudo unzip theme.zip -d /boot/grub/themes/mytheme/

# 3. 确认 theme.txt 文件存在
ls /boot/grub/themes/mytheme/theme.txt
```

### 4.3 启用主题

编辑 `/etc/default/grub`，添加或修改以下配置：

```bash
GRUB_THEME="/boot/grub/themes/mytheme/theme.txt"
GRUB_GFXMODE="1920x1080,auto"
```

然后更新配置：

```bash
sudo update-grub
```

### 4.4 单独设置背景图片

如果不想使用完整主题，也可以只设置一张背景图片：

```bash
# 1. 准备一张 PNG 格式图片（推荐 1920x1080，24位色深）
sudo cp background.png /boot/grub/

# 2. 在 /etc/default/grub 中添加
GRUB_BACKGROUND="/boot/grub/background.png"

# 3. 更新配置
sudo update-grub
```

### 4.5 使用 Grub Customizer 图形化工具

对于不习惯命令行的用户，`grub-customizer` 提供了直观的图形界面来管理主题和启动项。

```bash
# Ubuntu/Debian
sudo apt install grub-customizer
```

通过该工具可以轻松更换主题、调整分辨率、重命名菜单项等。

### 4.6 常见美化问题

- **主题不生效**：确认 `theme.txt` 文件格式正确，且路径拼写无误。更新配置后重启查看效果。
- **黑屏或字体错位**：部分显卡驱动（尤其是 NVIDIA）与高分辨率主题存在兼容问题。可在 `/etc/default/grub` 的 `GRUB_CMDLINE_LINUX_DEFAULT` 中添加 `fbcon=map:1` 参数，或降低分辨率。
- **查看支持的分辨率**：在 GRUB 启动菜单按 `c` 进入命令行，输入 `videoinfo` 查看可用分辨率。


## 五、GRUB2 安全加固

### 5.1 设置编辑权限密码

设置密码后，用户必须输入密码才能通过按 `e` 编辑启动项或按 `c` 进入命令行，有效防止未经授权的修改。

**方法一：使用 grub2-setpassword（最简单）**

```bash
sudo grub2-setpassword
# 按提示输入并确认密码
```

执行后会自动生成 `/boot/grub2/user.cfg` 文件，无需其他操作。

**方法二：手动配置（更灵活）**

```bash
# 1. 生成密码哈希
sudo grub2-mkpasswd-pbkdf2
# 复制输出的哈希字符串

# 2. 编辑 /etc/grub.d/40_custom
sudo nano /etc/grub.d/40_custom

# 添加以下内容：
set superusers="admin"
password_pbkdf2 admin grub.pbkdf2.sha512.10000.XXXXXXXXXXXX

# 3. 更新配置
sudo update-grub
```

### 5.2 限制特定菜单项需要密码

可以在自定义菜单项中添加 `--users` 参数，使特定系统启动时也需要密码验证：

```bash
menuentry 'Restricted System' --users admin {
    insmod ext2
    set root='hd0,gpt5'
    linux /boot/vmlinuz root=/dev/sda5
    initrd /boot/initrd.img
}
```

### 5.3 重要提醒

- 使用 `grub2-setpassword` 设置的密码只防止菜单编辑，**不防止未经授权的启动**。
- 务必妥善保管密码，忘记密码可能需要通过 Live CD 或救援模式才能恢复。


## 六、GRUB2 故障排查与修复

### 6.1 GRUB Rescue 救援模式

当 GRUB2 配置文件损坏或丢失时，系统会进入 `grub rescue>` 救援模式。此时可按以下步骤手动引导系统：

```bash
# 1. 查看可用的硬盘和分区
ls
# 输出如：(hd0) (hd0,gpt1) (hd0,gpt2) ...

# 2. 定位包含 /boot 目录的分区
ls (hd0,gpt2)/boot/
# 尝试不同分区，直到看到内核文件

# 3. 设置根分区和前缀
set root=(hd0,gpt2)
set prefix=(hd0,gpt2)/boot/grub

# 4. 加载正常模块
insmod normal
normal
```

如果 `normal` 模块丢失，则需要手动加载内核：

```bash
# 加载必要模块
insmod part_gpt
insmod ext2

# 设置内核和 initrd
linux /boot/vmlinuz-xxx root=/dev/sda2
initrd /boot/initrd.img-xxx

# 启动
boot
```

### 6.2 使用 Live CD/USB 修复 GRUB

当系统完全无法启动时，需要使用安装盘或 Live USB 进入救援环境进行修复。

```bash
# 1. 挂载系统根分区
sudo mount /dev/sda2 /mnt
sudo mount /dev/sda1 /mnt/boot/efi   # UEFI 系统

# 2. 挂载必要的虚拟文件系统
sudo mount --bind /dev /mnt/dev
sudo mount --bind /proc /mnt/proc
sudo mount --bind /sys /mnt/sys

# 3. chroot 进入系统环境
sudo chroot /mnt

# 4. 重新安装 GRUB
# BIOS 系统：
grub-install /dev/sda
# UEFI 系统：
grub-install --target=x86_64-efi --efi-directory=/boot/efi --bootloader-id=GRUB

# 5. 重新生成配置文件
grub-mkconfig -o /boot/grub/grub.cfg

# 6. 退出 chroot 并重启
exit
sudo reboot
```

### 6.3 常见错误与解决方案

| 错误信息 | 可能原因 | 解决方案 |
|---------|---------|---------|
| `error: file '/grub/i386-pc/normal.mod' not found` | GRUB2 模块文件丢失或损坏 | 使用 Live CD 重新安装 GRUB2 |
| `error: unknown filesystem` | 分区文件系统类型无法识别 | 检查分区是否正确，加载对应模块（如 `insmod ext2`） |
| `error: no such partition` | 分区 UUID 已更改（如调整分区后） | 进入救援模式手动设置 root 分区，进入系统后运行 `update-grub` |
| Windows 启动项消失 | `os-prober` 未安装或未启用 | 安装 `os-prober`，在 `/etc/default/grub` 中启用并更新配置 |

### 6.4 查看 GRUB 版本信息

```bash
grub-install --version
```

## 七、GRUB2 模块管理

GRUB2 的模块化设计使其核心保持精简，大多数功能（如特定文件系统支持、加密、图形等）都以模块形式存在，按需加载。

### 7.1 模块存储位置

模块文件通常存放在以下目录：

- BIOS 系统：`/boot/grub/i386-pc/`
- UEFI x86_64：`/boot/grub/x86_64-efi/`

文件扩展名为 `.mod`，如 `ext2.mod`、`ntfs.mod`、`luks.mod` 等。

### 7.2 常用模块与功能

| 模块名 | 功能描述 |
|--------|----------|
| `ext2` | ext2/ext3/ext4 文件系统支持 |
| `ntfs` | NTFS 文件系统读取 |
| `btrfs` | Btrfs 文件系统支持 |
| `xfs` | XFS 文件系统支持 |
| `part_gpt` | GPT 分区表支持 |
| `part_msdos` | MBR 分区表支持 |
| `lvm` | LVM 逻辑卷管理支持 |
| `luks` / `luks2` | LUKS 加密卷解锁 |
| `gfxterm` | 图形化终端（用于主题） |
| `gfxmenu` | 图形化菜单支持 |
| `video` | 视频模式设置 |
| `png` | PNG 图片背景支持 |
| `password_pbkdf2` | 密码验证功能 |
| `chain` | 链式加载其他引导程序 |

### 7.3 在命令行中手动加载模块

在 GRUB 命令行或自定义菜单项中，使用 `insmod` 加载模块：

```bash
insmod part_gpt
insmod ext2
insmod ntfs
```

### 7.4 查看已加载的模块

进入 GRUB 命令行（按 `c` 键），输入：

```bash
lsmod
```

### 7.5 模块缺失的修复

若出现 `error: file '/grub/i386-pc/normal.mod' not found`，说明核心模块丢失，需通过 Live 环境重新生成模块目录：

```bash
grub-install --target=i386-pc --boot-directory=/mnt/boot /dev/sda
grub-mkconfig -o /mnt/boot/grub/grub.cfg
```

## 八、GRUB2 命令行界面（GRUB Shell）深入使用

GRUB2 提供了一个功能强大的命令行解释器，可在启动菜单按 `c` 进入。它不仅用于故障恢复，也可用于测试、学习甚至临时启动任意系统。

### 8.1 基本导航命令

```bash
# 列出所有设备
ls

# 列出分区中的文件
ls (hd0,gpt2)/

# 列出特定目录
ls (hd0,gpt2)/boot/

# 查看当前设置的变量
set

# 设置变量（临时）
set root=(hd0,gpt2)
set timeout=10
```

### 8.2 查看分区 UUID 与文件系统标签

```bash
# 显示设备详细信息
ls -l
# 输出包含 UUID 和文件系统类型
```

### 8.3 手动启动 Linux 内核

当菜单项损坏时可临时手动启动：

```bash
# 加载必要模块
insmod part_gpt
insmod ext2

# 设置根分区（包含 /boot 的分区）
set root=(hd0,gpt2)

# 指定内核与 initrd（可用 Tab 键自动补全）
linux /boot/vmlinuz-5.15.0-91-generic root=/dev/sda2 ro quiet splash
initrd /boot/initrd.img-5.15.0-91-generic

# 启动
boot
```

### 8.4 链式加载其他 EFI 应用

```bash
insmod chain
set root=(hd0,gpt1)
chainloader /EFI/Microsoft/Boot/bootmgfw.efi
boot
```

### 8.5 常用环境变量说明

在 GRUB 命令行输入 `set` 可看到当前环境变量：

| 变量名 | 含义 |
|--------|------|
| `root` | 当前设备（如 `(hd0,gpt2)`） |
| `prefix` | 模块与配置文件的搜索路径 |
| `gfxmode` | 当前图形分辨率 |
| `lang` | 界面语言 |
| `timeout` | 菜单超时时间 |
| `default` | 默认菜单项 |

## 九、内核启动参数详解与临时修改

启动时按 `e` 编辑菜单项，可在 `linux` 行末尾添加或修改内核参数，用于调试、兼容性调整或性能优化。

### 9.1 常用调试参数

| 参数 | 作用 |
|------|------|
| `single` 或 `1` | 进入单用户维护模式 |
| `systemd.unit=rescue.target` | 进入 systemd 救援模式 |
| `init=/bin/bash` | 直接启动 Bash Shell（跳过 init） |
| `debug` | 输出详细内核调试信息 |
| `rd.debug` | 输出 initrd 阶段的调试信息 |

### 9.2 显示与兼容性参数

| 参数 | 作用 |
|------|------|
| `nomodeset` | 禁用内核模式设置，解决显卡驱动黑屏问题 |
| `nouveau.modeset=0` | 禁用 NVIDIA 开源驱动 |
| `acpi=off` | 禁用 ACPI（用于老旧硬件） |
| `noapic` | 禁用 APIC 中断控制器 |
| `irqpoll` | 轮询中断，解决中断分配问题 |
| `pci=noacpi` | 禁用 PCI 的 ACPI 管理 |
| `i915.modeset=0` | 禁用 Intel 集成显卡内核模式设置 |
| `radeon.modeset=0` | 禁用 AMD Radeon 开源驱动 |

### 9.3 内存与性能相关

| 参数 | 作用 |
|------|------|
| `mem=4G` | 限制系统可用内存大小 |
| `maxcpus=2` | 限制使用的 CPU 核心数 |
| `nosmp` | 禁用对称多处理（单核模式） |
| `mitigations=off` | 关闭 CPU 漏洞缓解措施（提升性能但降低安全） |

### 9.4 临时修改的注意事项

- 在 GRUB 菜单按 `e` 所做的修改**仅本次启动有效**，不会保存到配置文件中。
- 若要使参数永久生效，需在 `/etc/default/grub` 的 `GRUB_CMDLINE_LINUX_DEFAULT` 中添加对应参数，然后运行 `update-grub`。

## 十、initrd/initramfs 相关操作

initrd（Initial RAM Disk）是启动过程中内核加载的临时根文件系统，包含必要的驱动和工具，用于挂载真实根文件系统。

### 10.1 查看当前内核的 initrd 文件

```bash
ls /boot/initrd.img-*
# 或
ls /boot/initramfs-*.img
```

### 10.2 重新生成 initramfs

当磁盘控制器驱动变更或修改了 `/etc/mkinitcpio.conf`（Arch）或 `/etc/initramfs-tools/initramfs.conf`（Debian）后需要重新生成。

**Debian/Ubuntu：**
```bash
sudo update-initramfs -u -k all
```

**RHEL/CentOS/Fedora：**
```bash
sudo dracut --force --regenerate-all
```

**Arch Linux：**
```bash
sudo mkinitcpio -P
```

### 10.3 在 GRUB 中加载自定义 initrd

自定义菜单项示例：

```bash
menuentry 'Custom Kernel' {
    insmod ext2
    set root='hd0,gpt2'
    linux /boot/vmlinuz-custom root=/dev/sda2
    initrd /boot/initrd.img-custom
}
```

### 10.4 使用多个 initrd 文件

某些配置需要加载多个 initrd（如微码更新 + 主 initrd）：

```bash
initrd /boot/intel-ucode.img /boot/initrd.img-5.15.0
```

## 十一、配置文件的备份与恢复

### 11.1 备份关键文件

建议备份以下内容：

```bash
# 创建备份目录
mkdir ~/grub-backup

# 备份主配置目录
sudo cp -r /boot/grub ~/grub-backup/

# 备份用户配置
sudo cp /etc/default/grub ~/grub-backup/

# 备份脚本目录
sudo cp -r /etc/grub.d ~/grub-backup/

# 如果有自定义主题
sudo cp -r /boot/grub/themes ~/grub-backup/themes/
```

### 11.2 恢复配置

```bash
# 恢复用户设置
sudo cp ~/grub-backup/grub /etc/default/

# 恢复脚本目录
sudo cp -r ~/grub-backup/grub.d /etc/

# 恢复主题
sudo cp -r ~/grub-backup/themes /boot/grub/

# 重新生成 grub.cfg
sudo update-grub
```

### 11.3 使用版本控制管理配置

对于高级用户，可将 `/etc/default/grub` 和 `/etc/grub.d/` 置于 Git 仓库中：

```bash
cd /etc
sudo git init
sudo git add default/grub grub.d/
sudo git commit -m "Initial GRUB2 config"
```

之后每次修改都进行提交，方便追溯变更。

## 十二、从 GRUB Legacy 升级到 GRUB2

GRUB Legacy（版本 0.9x）与 GRUB2 的配置方式完全不同，升级需谨慎操作。

### 12.1 升级前检查

```bash
# 确认当前 GRUB 版本
grub-install --version
# 或
grub --version
```

### 12.2 升级步骤（以 Ubuntu 为例）

```bash
# 1. 安装 GRUB2 软件包
sudo apt install grub-pc

# 2. 跟随安装向导选择安装目标磁盘（通常为 /dev/sda）

# 3. 将 Legacy 配置文件转换为 GRUB2 格式（可选）
sudo upgrade-from-grub-legacy

# 4. 生成新配置文件
sudo update-grub

# 5. 写入引导扇区
sudo grub-install /dev/sda
```

### 12.3 清理旧版文件

升级成功并确认系统可正常启动后，可移除旧版软件包：

```bash
sudo apt purge grub-legacy
```

### 12.4 注意事项

- 务必在升级前备份 `/boot/grub/menu.lst`（Legacy 配置文件）
- 如果有 Windows 双系统，升级后需确保 `os-prober` 正常工作
- 升级后旧的 `menu.lst` 不再生效，所有配置需在 `/etc/default/grub` 中重新设定

## 十三、GRUB2 与 systemd-boot 的比较与切换

对于 UEFI 系统，systemd-boot（原 gummiboot）是一个轻量级替代方案。

### 13.1 主要区别

| 特性 | GRUB2 | systemd-boot |
|------|-------|--------------|
| 配置文件 | `/boot/grub/grub.cfg`（自动生成） | `/boot/loader/entries/*.conf`（手动编写） |
| 模块化 | 是，大量 `.mod` 文件 | 否，功能精简 |
| 支持 BIOS | 是 | 否（仅 UEFI） |
| 主题支持 | 丰富 | 有限（仅文本菜单） |
| 多系统探测 | 自动（os-prober） | 需手动编写条目 |
| 启动速度 | 较慢 | 极快 |

### 13.2 从 GRUB2 迁移到 systemd-boot

```bash
# 1. 安装 systemd-boot
sudo bootctl install

# 2. 创建启动条目（示例）
sudo mkdir -p /boot/loader/entries
sudo tee /boot/loader/entries/arch.conf << EOF
title   Arch Linux
linux   /vmlinuz-linux
initrd  /initramfs-linux.img
options root=PARTUUID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx rw
EOF

# 3. 配置 loader.conf
sudo tee /boot/loader/loader.conf << EOF
default arch
timeout 3
EOF

# 4. 移除 GRUB2（可选，但建议先保留双引导测试）
# sudo pacman -R grub  # Arch
# sudo apt purge grub-pc  # Debian
```

### 13.3 双引导共存

两种引导程序可同时存在于 ESP（EFI 系统分区），通过 UEFI 启动顺序选择使用哪一个：

```bash
# 查看当前启动项
efibootmgr -v

# 设置启动顺序（将 Boot0001 置前）
sudo efibootmgr -o 0001,0000
```

## 十四、高级主题定制技巧

### 14.1 自定义字体

GRUB2 支持 PF2 格式字体，可使用 `grub-mkfont` 工具转换系统字体：

```bash
# 转换 TTF 字体为 PF2
sudo grub-mkfont -s 16 -o /boot/grub/fonts/custom.pf2 /usr/share/fonts/TTF/DejaVuSans.ttf

# 在 theme.txt 中引用
echo 'font = "custom.pf2"' >> /boot/grub/themes/mytheme/theme.txt
```

### 14.2 修改菜单项图标对齐

在 `theme.txt` 中调整图标位置和大小：

```
+ boot_menu {
    left = 30%
    top = 25%
    width = 50%
    height = 50%
    item_font = "DejaVu Sans Regular 12"
    item_color = "#cccccc"
    selected_item_color = "#ffffff"
    item_height = 30
    item_spacing = 5
    icon_width = 32
    icon_height = 32
    item_icon_space = 10
}
```

### 14.3 添加动态进度条

部分主题支持显示启动倒计时进度条，在 `theme.txt` 中定义：

```
+ progress_bar {
    id = "__timeout__"
    left = 35%
    top = 80%
    width = 30%
    height = 10
    bar_style = "normal"
    highlight_style = "selected"
}
```

### 14.4 使用 grub-theme-preview 工具预览主题

无需重启即可预览主题效果：

```bash
# 安装
sudo apt install grub-imageboot   # Debian/Ubuntu

# 预览
grub-theme-preview /boot/grub/themes/mytheme/theme.txt
```

## 十五、常见问题补充解答

### 15.1 如何隐藏 GRUB 菜单直接启动？

编辑 `/etc/default/grub`：

```bash
GRUB_TIMEOUT=0
GRUB_TIMEOUT_STYLE=hidden
```

若要临时显示菜单，启动时按住 `Shift` 键（BIOS）或 `Esc` 键（UEFI）。

### 15.2 如何修改 GRUB 菜单的默认语言？

```bash
GRUB_LANG=zh_CN
```

需确保 `/boot/grub/locale/` 目录下有对应的 `zh_CN.mo` 文件，可通过 `grub-install` 时安装。

### 15.3 如何在 GRUB 中启用串行控制台？

编辑 `/etc/default/grub`：

```bash
GRUB_TERMINAL="serial"
GRUB_SERIAL_COMMAND="serial --speed=115200 --unit=0 --word=8 --parity=no --stop=1"
```

### 15.4 如何删除多余的内核启动项？

卸载旧内核后，运行 `sudo update-grub` 会自动清理菜单项。也可以手动在 `/etc/grub.d/` 中禁用自动探测脚本。

### 15.5 升级内核后 GRUB 菜单未更新？

正常情况下 `update-grub` 会在内核安装/卸载时被自动触发。若未更新，请手动执行一次，或检查 `/etc/kernel/postinst.d/zz-update-grub` 脚本是否存在。

```bash
sudo update-grub
```
