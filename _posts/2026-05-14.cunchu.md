## 1. 基本记忆单元：RS 锁存器

**RS 锁存器** 是使用交叉耦合门构成的最简双稳态电路。
~~（MC里可以使用两根红石火把实现）~~
以或非门实现为例，两个或非门的输出分别连接至对方的一个输入端，另两个输入端分别定义为 $S$（置位）和 $R$（复位）。输出端 $Q$ 与 $\overline{Q}$ 通常呈互补关系。

电路的行为可概括为：

- $S=0,\;R=0$：若此前 $Q=0$，则保持 $Q=0$；若 $Q=1$，保持 $Q=1$。此时电路处于存储状态。
- $S=1,\;R=0$：不论原状态如何，$Q$ 被置为 1（置位）。
- $S=0,\;R=1$：$Q$ 被清为 0（复位）。
- $S=1,\;R=1$：两输出均为 0，互补关系破坏；且当两输入同时回到 0 时，次态不可预测。该输入组合通常被禁止。

**RS 锁存器无法直接用于需要统一时序的同步系统。**

## 2. 同步化：D 锁存器与 D 触发器

### 2.1 D 锁存器

为避免 RS 锁存器的禁止态，可将 $S$ 和 $R$ 改造为互补控制。在 RS 锁存器前增加两与门及一个反相器，引入数据输入端 $D$ 和使能端 $E$。当 $E=1$ 时，$S = D$，$R = \overline{D}$，内部锁存器置位或复位；当 $E=0$ 时，两与门输出均为 0，锁存器保持。该电路称为 **D 锁存器**（Data Latch）。

```plaintext
          +-----+
 D ───────┤ AND ├──── S ──┐
          +--+--+         │
             │            │
 E ──────────┘            │
                          │
          +-----+         │
 D ──┬────┤ NOT ├───┐     │
     │    +-----+   │     │
     │              │     │
     │    +-----+   │     │
     └────┤ AND ├─── R ──┘
          +--+--+       RS 锁存器
             │
 E ──────────┘

 RS 锁存器输出：Q, Qn
 ```

### 2.2 D 触发器

*参考：Wiki*

将两级 D 锁存器级联构成**主从 D 触发器**。第一级主锁存器由时钟信号 $CLK$ 使能，第二级从锁存器由 $\overline{CLK}$ 使能。当时钟为高电平时，主锁存器透明，从锁存器保持；时钟下降沿来临时，主锁存器锁存当前 $D$ 值，从锁存器将其传递至输出 $Q$。最终输出仅在时钟边沿更新，实现了边沿触发存储。

```plaintext
       ┌──────────┐       ┌──────────┐
  D ───┤ 主锁存器 ├── P ──┤ 从锁存器 ├── Q
       │ (D锁存器) │       │ (D锁存器) │
       └────┬─────┘       └────┬─────┘
            │                  │
       CLK ─┘                  │
                           ┌───┴───┐
                           │  NOT  │
                           └───┬───┘
                               │
                              ~CLK
```

## 3. 寄存器与寄存器组

### 3.1 寄存器

将多个 D 触发器并接，共用同一时钟，可构成多位的**寄存器**。每个触发器的 $D$ 端接收一位数据，$Q$ 端输出，所有位在同一时钟边沿更新。寄存器用于暂存 CPU 内部的操作数、地址和状态字，具备单周期读写能力，但面积和功耗限制了其数量。

### 3.2 寄存器组

为满足处理器同时对多个寄存器进行读取和写入的需求，将若干个寄存器组织成**寄存器组**。

典型的双读单写寄存器组结构如下：

- 由译码器将写地址转换为独热码，选中目标寄存器使能写时钟。
- 两个多路选择器分别由两个读地址控制，从所有寄存器输出中选择对应的数据送至读端口。

这样，可在同一时钟周期内完成两个读操作和一个写操作，支持指令流水线中的并行数据访问。

## 4. 静态随机存取存储器（SRAM）与内存阵列

*参考：《计算机组成原理》第三版 （唐朔飞教授编写），CSDN，Deepseek*

### 4.1 SRAM 单元

大面积存储需要更紧凑的单元。**6 管 SRAM 单元**由两个交叉耦合的反相器构成存储核心，两个 NMOS 存取晶体管连接至互补位线对（BL 与 $\overline{BL}$），栅极受字线（WL）控制。

- 保持状态：WL 为低，存取管关断，反相器环维持电平。
- 读操作：位线预充至逻辑高，WL 开启后存储节点与位线连通，存储“0”的一侧轻微拉低位线，灵敏放大器检测位线间差分并放大至全摆幅。读过程不破坏单元内容。
- 写操作：将欲写入数据强加在位线对上，开启 WL，外部驱动强度高于单元内反相器，强制翻转存储状态。

### 4.2 存储阵列组织

SRAM 单元按行列矩阵排列。行译码器根据地址高位激活一条字线，选中整行单元；列译码器通过多路选择器从被选行的位线中选出所需位宽的数据，经灵敏放大器后输出或接收写入数据。行列译码结构可将地址线数从 $2^N$ 级减少至译码器规模，实现高密度存储。

### 4.3 DRAM 概述

动态随机存取存储器（DRAM）单元由一个晶体管和一个电容构成（1T1C），利用电容电荷表示数据。因漏电需定期刷新，读操作具有破坏性，需自动回写。DRAM 采用分时复用的地址总线（行地址先锁存，再列地址），密度较 SRAM 更高，成本更低，作为计算机主存使用。

## 5. 栈

**栈**是一种后进先出的存储区域，通常由一块普通内存与**栈指针（SP）**寄存器配合实现。栈向低地址方向增长的操作常见于多数架构。

- 压栈（PUSH）：`SP ← SP – 操作数宽度`，然后将数据写入 `MEM[SP]`。
- 弹栈（POP）：读出 `MEM[SP]` 的数据，然后 `SP ← SP + 操作数宽度`。

CALL 指令执行时自动将返回地址压入栈中，RET 指令从栈中弹出地址返回。部分嵌入式处理器内置深度有限的硬件返回栈，以寄存器堆方式实现，不需占用外部总线，但需软件保证不溢出。

## 6. 堆空间与动态内存分配

在程序地址空间中，**堆**位于静态数据区与栈之间，向高地址增长。动态内存分配（如 C 语言的 `malloc` 和 `free`）由堆管理器负责，不属于硬件强制功能，而是由运行时库或操作系统内核实现的资源管理策略。

堆管理器通常维护一个或一组**空闲块链表**。每个空闲块头部包含大小、状态及指向下一空闲块的指针。

- **分配**：遍历链表，按首次适配、最佳适配等策略查找足够大的空闲块；若块远大于请求尺寸，则分割为两块，剩余部分重新加入链表；返回分配区指针。
- **释放**：根据返回指针定位块头，标记为空闲；检查相邻物理块是否空闲，若空闲则合并，以减少外部碎片。

在硬实时或资源受限系统中，也存在硬件加速的固定大小分区分配器，通过位图查找空闲块，分配与释放时间为常数，适用于网络缓冲、任务控制块等场景。
