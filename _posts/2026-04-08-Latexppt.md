## 一、Beamer 文档的基本骨架

### 1.1 文档类声明

每一个 Beamer 文档都以 `\documentclass` 开头，并可通过可选参数定制全局设置：

```latex
\documentclass[aspectratio=169, utf8, 12pt]{beamer}
```

常用类选项：
- `aspectratio`：幻灯片宽高比，可选 `43`（4:3）、`169`（16:9）、`1610`（16:10）等。
- `draft`：草稿模式，图片以占位框显示，加快编译速度。
- `compress`：压缩导航栏间距，增加内容区域。
- `handout`：生成讲义版本，忽略所有动画覆盖。

### 1.2 导言区：宏包与全局设置

在 `\begin{document}` 之前的部分称为导言区，用于加载宏包和进行全局配置。

```latex
\usepackage{ctex}           % 中文支持
\usepackage{amsmath, amssymb} % 数学扩展
\usepackage{graphicx}       % 插图
\usepackage{booktabs}       % 三线表
\usepackage{listings}       % 代码高亮

\title[短标题]{完整的演示文稿标题}
\subtitle{副标题（可选）}
\author[报告人]{作者姓名\inst{1} \and 合作者\inst{2}}
\institute[单位简称]{\inst{1}第一单位 \\ \inst{2}第二单位}
\date{\today}
\logo{\includegraphics[height=0.8cm]{logo.pdf}} % 全局 Logo
```

标题、作者等信息既可在导言区定义，也可在正文的 `\titlepage` 前定义。中括号内为短标题版本，将显示在导航栏或页脚。

### 1.3 正文结构：frame 环境

Beamer 的内容单位是 **帧（frame）**，每一帧对应一页幻灯片。

```latex
\begin{document}

\begin{frame}
  \titlepage
\end{frame}

\begin{frame}{帧标题}{帧副标题（可选）}
  内容...
\end{frame}

\end{document}
```

帧环境支持可选参数：
- `[plain]`：去除页眉页脚和导航栏，只保留内容。
- `[fragile]`：当帧内包含 `verbatim` 或 `listings` 环境时必须使用。
- `[allowframebreaks]`：当内容超过一页时自动分页（谨慎使用，通常不推荐）。

## 二、中文排版

Beamer 处理中文的关键在于**编译器选择**与**宏包配置**。

### 2.1 组合：XeLaTeX + ctex

**务必使用 XeLaTeX 编译器**。它原生支持 UTF-8 编码和系统字体，是中文 LaTeX 的最优解。

在 Overleaf 中：点击 Menu → Compiler → 选择 **XeLaTeX**。  
在本地编辑器中：在文件首行添加 `% !TEX program = xelatex` 以指定编译器。

随后，在导言区加入一行：

```latex
\usepackage{ctex}
```

`ctex` 宏包会自动配置中文字体、字号和段落缩进，无需额外操作即可获得正确的中文显示。

### 2.2 自定义中文字体（可选）

若对默认字体（Windows 下为中易宋体/黑体）不满意，可手动指定：

```latex
\usepackage{fontspec}
\setCJKmainfont{Noto Serif CJK SC}      % 思源宋体
\setCJKsansfont{Noto Sans CJK SC}       % 思源黑体
\setCJKmonofont{FangSong}               % 仿宋
```

## 三、内容元素：从文字到公式的完整语法

### 3.1 文本块与段落

Beamer 帧内直接书写文本即可，段落之间用空行分隔。

```latex
\begin{frame}{纯文本示例}
这是第一段。

这是第二段，Beamer 默认不会自动缩进首行，但会保留段间距。
\end{frame}
```

**强调文本：**

```latex
\emph{强调} \textbf{粗体} \textit{斜体} \texttt{等宽字体}
```

### 3.2 列表环境

Beamer 支持三种列表：无序列表 `itemize`、有序列表 `enumerate` 和描述列表 `description`。

```latex
\begin{frame}{列表示例}
  \begin{itemize}
    \item 第一项
    \item 第二项
    \begin{itemize}
      \item 嵌套子项
    \end{itemize}
  \end{itemize}

  \begin{enumerate}
    \item 步骤一
    \item 步骤二
  \end{enumerate}

  \begin{description}
    \item[术语] 定义说明
    \item[Beamer] 一种 LaTeX 演示文档类
  \end{description}
\end{frame}
```

列表环境可与覆盖动画结合，详见后文。

### 3.3 分栏布局：columns 环境

分栏是学术幻灯片中最常用的排版手段之一。

```latex
\begin{frame}{分栏布局}
  \begin{columns}[T]  % T 表示顶部对齐，也可用 c（居中）、b（底部）
    \begin{column}{0.48\textwidth}
      左侧内容，可以包含文字、公式、列表等。
    \end{column}
    \begin{column}{0.48\textwidth}
      \includegraphics[width=\textwidth]{figure.pdf}
    \end{column}
  \end{columns}
\end{frame}
```

> **注意**：在 `columns` 环境中，总宽度之和应略小于 `\textwidth`，留出栏间距。`[T]` 对齐选项能避免图文顶部参差不齐。

### 3.4 插图与表格

**插入图片：**

```latex
\begin{frame}{插图示例}
  \begin{figure}
    \centering
    \includegraphics[width=0.6\textwidth]{image.png}
    \caption{这是一张图片的说明文字}
    \label{fig:example}
  \end{figure}
\end{frame}
```

> ⚠️ 警告：在某些 Beamer 版本中，`figure` 浮动体可能导致编译错误或内容溢出。更稳妥的做法是直接使用 `\includegraphics` 并配合 `\centering` 和 `\captionof{figure}{...}`（需 `caption` 宏包）。

**制作表格：**

推荐使用 `booktabs` 宏包绘制三线表，简洁专业。

```latex
\usepackage{booktabs}

\begin{frame}{三线表示例}
  \begin{table}
    \centering
    \begin{tabular}{lcc}
      \toprule
      方法 & 准确率 & 召回率 \\
      \midrule
      Baseline & 0.82 & 0.79 \\
      Ours     & 0.91 & 0.88 \\
      \bottomrule
    \end{tabular}
    \caption{实验结果对比}
  \end{table}
\end{frame}
```

### 3.5 数学公式与定理环境

Beamer 完全继承 LaTeX 的数学排版能力。

**行内公式：** `$E = mc^2$`  
**行间公式：**
```latex
\[
  \int_{0}^{\infty} e^{-x^2} dx = \frac{\sqrt{\pi}}{2}
\]
```

**带编号的公式：**
```latex
\begin{equation}
  \hat{\beta} = (X^T X)^{-1} X^T y
  \label{eq:ols}
\end{equation}
```

**定理类环境：** Beamer 预定义了 `theorem`、`definition`、`example`、`proof` 等环境，样式与主题匹配。

```latex
\begin{frame}{定理环境示例}
  \begin{theorem}[勾股定理]
    直角三角形两直角边的平方和等于斜边的平方：
    \[ a^2 + b^2 = c^2 \]
  \end{theorem}

  \begin{proof}
    如图，构造四个全等直角三角形拼成正方形...
  \end{proof}

  \begin{definition}[机器学习]
    机器学习是人工智能的一个分支，研究计算机如何从数据中学习规律。
  \end{definition}
\end{frame}
```

### 3.6 代码块展示

使用 `listings` 宏包可实现语法高亮。注意需在帧环境后添加 `[fragile]` 选项。

```latex
\begin{frame}[fragile]{代码展示示例}
  \begin{lstlisting}[language=Python, caption={冒泡排序}]
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(0, n-i-1):
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]
  \end{lstlisting}
\end{frame}
```

### 3.7 块环境（Block）

块用于突出显示重要内容，有三种预定义类型：普通块 `block`、示例块 `exampleblock` 和警示块 `alertblock`。

```latex
\begin{frame}{块环境}
  \begin{block}{普通块标题}
    这是普通块的内容，背景色与主题主色一致。
  \end{block}

  \begin{alertblock}{警示块标题}
    警示块通常用于强调重要结论，背景为红色系。
  \end{alertblock}

  \begin{exampleblock}{示例块标题}
    示例块背景为绿色系，适合展示应用案例。
  \end{exampleblock}
\end{frame}
```

## 四、动画与逐项显示：覆盖（Overlay）语法精讲

Beamer 的动画基于 **覆盖规范（Overlay Specification）**，通过尖括号 `< >` 控制内容出现的时机。

### 4.1 基础命令：`\pause`

最简单的逐项显示命令，在需要暂停的位置插入即可。

```latex
\begin{frame}{使用 pause}
  第一行内容 \pause
  第二行内容 \pause
  第三行内容
\end{frame}
```

每插入一个 `\pause`，编译后的 PDF 就会多一页，播放时通过翻页实现“点击出现”。

### 4.2 列表的自动覆盖：`[<+->]`

列表环境支持自动为每个条目分配覆盖序号。

```latex
\begin{frame}{列表自动覆盖}
  \begin{itemize}[<+->]
    \item 第一条
    \item 第二条
    \item 第三条
  \end{itemize}
\end{frame}
```

也可局部指定覆盖序号：
```latex
\begin{itemize}
  \item<1-> 第一步就出现，并保持到结束
  \item<2-> 第二步出现
  \item<3>  只在第三步出现，随后消失
  \item<1,3> 第一步和第三步出现
  \item<2-4> 第二步到第四步出现
\end{itemize}
```

### 4.3 精准控制：`\onslide` 与 `\only`

- `\onslide<n->{...}`：内容从第 n 步开始显示，之前的步骤中内容不可见但占据空间（占位）。
- `\only<n>{...}`：内容只在第 n 步显示，其他步骤完全不占据空间。

```latex
\begin{frame}{onslide 与 only 的区别}
  始终可见的文字。

  \onslide<2->{从第 2 步开始出现，占位。}

  \only<3>{只在第 3 步出现，不占位。}

  \onslide<1,3>{第 1 步和第 3 步出现，占位。}
\end{frame}
```

### 4.4 覆盖与公式、图片的配合

覆盖可以作用于任何 LaTeX 元素。

**公式逐步推导：**
```latex
\begin{frame}{公式逐步呈现}
  \begin{align*}
    f(x) &= x^2 + 2x + 1 \\
    \onslide<2->{&= (x+1)^2 \\}
    \onslide<3->{&\geq 0}
  \end{align*}
\end{frame}
```

**图片交替出现：**
```latex
\begin{frame}{图片交替}
  \only<1>{\includegraphics[width=0.8\textwidth]{fig1.pdf}}
  \only<2>{\includegraphics[width=0.8\textwidth]{fig2.pdf}}
\end{frame}
```

### 4.5 覆盖的高级语法

- `\uncover<n->{...}`：与 `\onslide` 类似，但不改变文本颜色。
- `\visible<n->{...}`：与 `\onslide` 几乎相同。
- `\invisible<n->{...}`：指定步骤中内容不可见。
- `\alt<n>{在步骤 n 显示的版本}{其他步骤显示的版本}`：条件分支。

```latex
\alt<2>{第二步时显示此文本}{非第二步时显示此文本}
```

## 五、主题与视觉定制

### 5.1 主题体系

Beamer 主题分为五个维度，每个维度可独立更换：

- **Presentation Theme**（整体主题）：`\usetheme{Madrid}`
- **Color Theme**（色彩主题）：`\usecolortheme{beaver}`
- **Font Theme**（字体主题）：`\usefonttheme{professionalfonts}`
- **Inner Theme**（内部元素：标题、列表、块）：`\useinnertheme{circles}`
- **Outer Theme**（外部元素：页眉页脚、侧边栏）：`\useoutertheme{infolines}`

**常用内置主题速览：**

| 主题名 | 特点 |
|-------|------|
| `default` | 极简，无任何装饰 |
| `Madrid` | 顶部导航栏，简洁大方 |
| `CambridgeUS` | 经典红灰，带有三行信息栏 |
| `Boadilla` | 干净清爽，无侧边栏 |
| `Pittsburgh` | 右侧带导航条 |
| `metropolis` | 现代极简，底部进度条（需安装） |

### 5.2 Metropolis 主题的深度定制

Metropolis 是近年来最受欢迎的第三方主题，以其克制、现代的设计著称。

```latex
\usetheme{metropolis}
\metroset{
  titleformat=smallcaps,      % 标题小型大写
  progressbar=frametitle,     % 进度条显示在标题栏
  block=fill,                 % 块背景填充颜色
  numbering=fraction          % 页码显示为 3/10 形式
}
```

### 5.3 自定义颜色

通过 `\setbeamercolor` 可精细调整每个元素的前景色和背景色。

```latex
\definecolor{myblue}{RGB}{0, 85, 150}
\definecolor{mygold}{RGB}{200, 150, 0}

\setbeamercolor{structure}{fg=myblue}                % 全局结构色
\setbeamercolor{frametitle}{bg=myblue, fg=white}     % 帧标题
\setbeamercolor{palette primary}{bg=myblue, fg=white}% 导航栏主色
\setbeamercolor{block title}{bg=myblue!80, fg=white}
\setbeamercolor{block body}{bg=myblue!10}
```

### 5.4 隐藏导航栏与页脚

有时需要极简的全屏展示，可使用 `plain` 帧选项，或全局禁用导航栏：

```latex
\setbeamertemplate{navigation symbols}{}  % 隐藏右下角导航图标
\setbeamertemplate{footline}[default]     % 可自定义或清空
```

## 六、组织大型文档：章节与目录

### 6.1 章节结构

Beamer 使用标准的 LaTeX 章节命令，这些命令不会生成新的幻灯片，但会向导航栏和目录提供信息。

```latex
\section{引言}
\subsection{研究背景}
\subsection{相关工作}

\section{方法}
\subsection{模型架构}
\subsection{训练细节}
```

### 6.2 目录页与节间引导页

```latex
\begin{frame}{目录}
  \tableofcontents
\end{frame}

% 在每一节开始前自动插入节标题页
\AtBeginSection[]{
  \begin{frame}{本节概要}
    \tableofcontents[currentsection, hideothersubsections]
  \end{frame}
}
```

`\tableofcontents` 可接受选项：
- `currentsection`：只显示当前节
- `hideothersubsections`：隐藏其他节的子节
- `pausesections`：逐节显示目录

## 七、编译与调试技巧

### 7.1 编译流程

Beamer 文档需经多次编译以正确生成交叉引用、目录和导航信息。推荐顺序：
```bash
xelatex presentation.tex
xelatex presentation.tex
```
两次编译后目录页码和超链接跳转才会正确。

### 7.2 加速编译

对于大型文档，草稿模式可显著提速：
```latex
\documentclass[draft]{beamer}  % 图片不实际载入
```
或临时注释掉图片和复杂表格，待定稿后再恢复。

### 7.3 常见错误速查

| 错误信息 | 可能原因 | 解决办法 |
|---------|---------|---------|
| `Missing \begin{document}` | 导言区有正文内容 | 检查 `\begin{document}` 之前是否有文字或空行 |
| `Not in outer par mode` | 帧内使用了浮动体 | 移除 `figure`/`table` 环境，或改用 `[H]` 选项 |
| 中文乱码 | 编译器非 XeLaTeX | 切换至 XeLaTeX 编译 |
| `Undefined control sequence` | 拼写错误或缺少宏包 | 检查命令名，确认已加载所需宏包 |
| 帧溢出警告 | 内容超出页面底部 | 拆分内容或使用 `allowframebreaks`（慎用） |

## 八、示例

### 8.1 逐步揭示复杂公式

```latex
\begin{frame}{贝叶斯公式推导}
  \begin{align*}
    P(A|B) &\onslide<1->{= \frac{P(B|A)P(A)}{P(B)} \\}
    \onslide<2->{&= \frac{P(B|A)P(A)}{\sum_i P(B|A_i)P(A_i)} \\}
    \onslide<3->{&\propto P(B|A)P(A)}
  \end{align*}
  \onslide<4->{其中最后一步利用了分母对所有假设恒定的性质。}
\end{frame}
```

### 8.2 并列对比两个图表

```latex
\begin{frame}{方法对比}
  \begin{columns}[T]
    \begin{column}{0.48\textwidth}
      \centering
      \includegraphics[width=\textwidth]{baseline.pdf}
      \textbf{基线方法}
    \end{column}
    \begin{column}{0.48\textwidth}
      \centering
      \includegraphics[width=\textwidth]{ours.pdf}
      \textbf{本文方法}
    \end{column}
  \end{columns}
  \vspace{1em}
  \only<2>{\alert{本文方法在边缘细节上提升明显。}}
\end{frame}
```

### 8.3 展示伪代码

```latex
\usepackage{algorithm, algorithmic}

\begin{frame}[fragile]{算法流程}
  \begin{algorithm}[H]
    \caption{梯度下降法}
    \begin{algorithmic}[1]
      \REQUIRE 初始参数 $\theta_0$，学习率 $\eta$
      \FOR{$t = 1$ to $T$}
        \STATE 计算梯度 $g_t = \nabla f(\theta_{t-1})$
        \STATE 更新 $\theta_t = \theta_{t-1} - \eta g_t$
      \ENDFOR
      \RETURN $\theta_T$
    \end{algorithmic}
  \end{algorithm}
\end{frame}
```

### 实例一：最简学术报告模板

这是一个适合组会汇报的极简模板，包含标题页、目录、分节和常见元素。

```latex
% !TEX program = xelatex
\documentclass[aspectratio=169]{beamer}
\usepackage{ctex}

% ---------- 基本信息 ----------
\title{基于深度学习的图像分割方法研究}
\subtitle{组会汇报}
\author{张三}
\institute{某某大学 \ 计算机学院}
\date{\today}

% ---------- 主题设置 ----------
\usetheme{Madrid}
\usecolortheme{beaver}

\begin{document}

% 标题页
\begin{frame}
  \titlepage
\end{frame}

% 目录页
\begin{frame}{目录}
  \tableofcontents
\end{frame}

% ---------- 第一节：引言 ----------
\section{引言}

\begin{frame}{研究背景}
  \begin{itemize}
    \item 图像分割是计算机视觉的基础任务之一
    \item 传统方法依赖手工特征，泛化能力有限
    \item 深度学习显著提升了分割精度，但仍面临挑战
  \end{itemize}
\end{frame}

\begin{frame}{相关工作}
  \begin{block}{经典模型}
    \begin{itemize}
      \item FCN：首个端到端的全卷积分割网络
      \item U-Net：编码器-解码器结构，广泛用于医学图像
      \item DeepLab：引入空洞卷积与条件随机场
    \end{itemize}
  \end{block}
  \pause
  \begin{alertblock}{现存问题}
    小目标分割精度不足，边界细节容易丢失。
  \end{alertblock}
\end{frame}

% ---------- 第二节：方法 ----------
\section{方法}

\begin{frame}{整体框架}
  \centering
  \includegraphics[width=0.8\textwidth]{example-image}  % 替换为你的图片
  \vspace{0.5em}

  网络采用编码器-解码器结构，并引入注意力机制增强特征表达。
\end{frame}
```

```latex
  \begin{definition}[效用值]
    项 $i$ 在事务 $T$ 中的效用值定义为：
    \[ u(i, T) = q(i, T) \times p(i) \]
    其中 $q(i, T)$ 为购买数量，$p(i)$ 为单位利润。

      \STATE 扫描数据库，计算所有项的 TWU
      \STATE 移除 TWU 小于阈值的项
      \STATE 构建效用列表结构
      \WHILE{效用列表集合非空}
        \STATE 取出一个项集的效用列表
        \STATE 计算其精确效用值
        \IF{效用值 $\ge$ 阈值}
          \STATE 输出该项集
        \ENDIF
        \STATE 与其余效用列表两两相交生成超集
      \ENDWHILE
    \end{algorithmic}
  \end{algorithm}
\end{frame}

\begin{frame}[fragile]{关键代码实现}

def construct_utility_list(itemset):
```
