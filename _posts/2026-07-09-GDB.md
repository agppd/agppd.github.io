## 一、准备工作

### 1.1 记得加 `-g`

Linux 下 `gcc`/`g++` 默认生成的是 **release 模式**的程序，不包含调试信息（如行号、变量名、类型信息等），无法用 GDB 调试。要使用 GDB，必须在编译时添加 `-g` 选项，生成 **debug 模式**的程序。

```bash
# release 模式（默认，无法调试）
gcc program.c -o program

# debug 模式（添加 -g，可调试）
gcc -g program.c -o program
```

如果未加 `-g` 选项，使用 GDB 调试时会提示无法识别调试信息。另外，调试优化后的代码可能会遇到变量不可见等问题，建议在调试时使用 `-O0` 关闭优化。

### 1.2 安装 GDB

在主流 Linux 发行版上，可以通过包管理器快速安装：

```bash
# Ubuntu/Debian
sudo apt-get install gdb

# CentOS/RHEL
sudo yum install gdb

# Fedora
sudo dnf install gdb
```

或者从源代码安装（[点我下载](https://ftp.gnu.org/gnu/gdb/gdb-17.2.tar.xz)）

```bash
cd CODE_SOURCES
./configure
make
mak install
```

## 二、启动与退出 GDB

### 2.1 启动方式

GDB 有三种常见的启动方式：

| 启动方式 | 命令 | 适用场景 |
|---------|------|---------|
| 直接调试 | `gdb ./program` | 调试普通的可执行程序 |
| 带参数启动 | `gdb --args ./program arg1 arg2` | 程序需要命令行参数 |
| 附加到运行进程 | `gdb -p <PID>` | 调试正在运行的进程（需权限） |
| 调试 core 文件 | `gdb ./program core` | 分析程序崩溃时的内存状态 |

启动成功后，会进入 GDB 交互界面，提示符为 `(gdb)`。

### 2.2 退出 GDB

退出 GDB 有两种方式：
- 输入 `quit`（或简写 `q`）并回车
- 按下 `Ctrl + d` 组合键

## 三、核心调试命令

### 3.1 查看源代码：`list`

调试时需要随时查看源代码以确定调试位置。`list`（可简写为 `l`）命令用于显示源代码：

```gdb
(gdb) list         
(gdb) list 20      
(gdb) list main    
(gdb) list test.c:8
```

`list` 命令具有记忆性，连续按回车会继续显示后续代码。

### 3.2 运行与控制

| 命令 | 简写 | 功能 |
|------|------|------|
| `run` | `r` | 启动程序（可带参数，如 `run 10 20`） |
| `continue` | `c` | 继续执行到下一个断点或程序结束 |
| `next` | `n` | 单步执行，**不进入**函数内部（相当于 VS 的 F10） |
| `step` | `s` | 单步执行，**进入**函数内部（相当于 VS 的 F11） |
| `finish` | — | 执行完当前函数并返回 |
| `until` | — | 跳到指定行 |

注意：GDB 会自动记录上一条输入指令，直接按回车会重复执行上一条命令——在单步调试时非常方便。

### 3.3 断点管理

断点是 GDB 调试的核心功能。

#### 设置断点

```gdb
(gdb) break 10         
(gdb) break main       
(gdb) break test.c:15  
```

`break` 可以简写为 `b`。

#### 查看与管理断点

```gdb
(gdb) info break       
(gdb) delete 2         
(gdb) disable 1        
(gdb) enable 1         
```

断点编号可以通过 `info break` 查看。

### 3.4 查看变量与表达式

#### 打印变量

```gdb
(gdb) print i          
(gdb) print *ptr       
(gdb) print arr@10     
```

#### 自动显示变量

`display` 命令可以在每次程序暂停时自动打印变量的值，非常适合持续观察某个变量：

```gdb
(gdb) display sum      
(gdb) info display     
(gdb) undisplay 1      
```

#### 查看变量类型

```gdb
(gdb) ptype variable   
```

### 3.5 查看调用栈

当程序在断点处暂停时，可以使用 `backtrace`（简写 `bt`）查看函数调用栈，快速了解程序的执行路径：

```gdb
(gdb) bt
#0  f () at test.c:3
#1  0x0000555555555172 in main () at test.c:10
```

注意：调用栈是**从下往上**看的——先调用 main，再调用 f。可以使用 `frame`（简写 `f`）切换栈帧。

### 3.6 查看局部变量

```gdb
(gdb) info locals      
```

## 四、高级调试技巧

### 4.1 条件断点

条件断点仅在满足特定条件时才触发，避免在无关的迭代中反复暂停：

```gdb
(gdb) break 10 if i == 5       
(gdb) break calculate if result > 1000  
```

对于已存在的断点，也可以用 `condition` 命令添加条件：

```gdb
(gdb) condition 2 i == 0       
```

### 4.2 观察点（Watchpoint）

观察点用于监控变量或内存的变化——当被监控的值被修改时，程序会自动暂停：

```gdb
(gdb) watch variable_name      
(gdb) rwatch variable_name     
(gdb) awatch variable_name     
```

这在追踪“这个变量到底在哪里被改了”时特别有用。

### 4.3 查看内存

`x` 命令用于按指定格式查看内存内容：

```gdb
(gdb) x/4xw &var       
(gdb) x/10cb buffer    
```

格式说明：`/<count><format><size>`，其中 format 包括 `x`（十六进制）、`d`（十进制）、`s`（字符串）、`i`（汇编指令）等，size 包括 `b`（字节）、`h`（半字）、`w`（字）、`g`（8字节）。

### 4.4 多线程调试

GDB 提供了完善的多线程调试支持：

```gdb
(gdb) info threads             
(gdb) thread 2                 
(gdb) thread apply all bt      
(gdb) set scheduler-locking on 
```

线程号前有 `*` 的表示当前正在调试的线程。

### 4.5 修改变量值

在调试过程中，可以直接修改变量的值来测试不同的执行路径：

```gdb
(gdb) set var i = 10           
```

### 4.6 TUI 文本界面

GDB 自带 TUI（Text User Interface）模式，可以同时显示源代码和命令窗口”：

```bash
gdb -tui ./program     
```

在 TUI 模式下的常用快捷键：
- `Ctrl + X` 再按 `A`：切换进入/退出 TUI 模式
- `Ctrl + X` 再按 `O`：切换焦点（代码窗口 ↔ 命令窗口）
- `Ctrl + L`：刷新屏幕（解决花屏问题）

### 4.7 脚本自动化

GDB 支持脚本功能，可以将常用的调试命令写入文件批量执行：

```bash
# debug.gdb 脚本内容
file program
break main
run
```

执行脚本：
```bash
gdb -x debug.gdb
```

### 4.8 配合 `-fsanitize=address` 一同调试

如如下代码

```cpp
#include<iostream>
#include<queue>
using namespace std;
const int N = 2e5 + 10;
int a[N];

priority_queue<pair<int, int>> q;

void solve()
{
    int n, x;
    cin >> n >> x;
    for (int i = 1; i <= n; i++)
        cin >> a[i];
    q.push({x + 1, 1});
    for (int i : a)
    {
        while (!q.empty() and q.top().first > i)
        {
            auto [val, idx] = q.top();
            q.pop();
            while (!q.empty() and q.top().first == val)
            {
                idx += q.top().second;
                q.pop();
            }
            q.push({i, val / i * idx});
            if (val % i != 0)
                q.push({val % i, idx});
        }
    }
    return ;
}
```

在输入
```plaintext
3 7
5 2 3
```
时，发生浮点异常。此时，我们猜测：应该是存在除0的情况（毕竟没有小数）。

所以：编译时我们加上 `-fsanitize=address` 参数，输出如下

## 五、命令缩写与快捷键

GDB 允许命令缩写，只要缩写没有歧义即可。

### 常用命令缩写

| 完整命令 | 缩写 | 功能 |
|----------|------|------|
| `run` | `r` | 运行程序 |
| `break` | `b` | 设置断点 |
| `continue` | `c` | 继续执行 |
| `next` | `n` | 单步执行（不进入函数） |
| `step` | `s` | 单步执行（进入函数） |
| `print` | `p` | 打印变量值 |
| `backtrace` | `bt` | 查看调用栈 |
| `quit` | `q` | 退出 GDB |
| `list` | `l` | 查看源代码 |
| `info` | `i` | 查看信息（如 `i b` 查看断点） |
| `delete` | `d` | 删除断点 |

### 特殊缩写规则

- `s` 是 `step` 的专属缩写，即使有其他 `s` 开头的命令（如 `set`、`show`），GDB 也规定 `s` 专指 `step`
- `i` 是 `info` 的合法缩写，因为没有任何其他命令以 `i` 开头
- 如果缩写有歧义（如 `in` 同时匹配 `info` 和 `inspect`），GDB 会提示错误

### 交互快捷键

| 快捷键 | 功能 |
|--------|------|
| `Tab` | 自动补全命令或符号名 |
| `↑` / `↓` | 浏览历史命令 |
| `Enter`（空白行） | 重复上一条命令 |
| `Ctrl + R` | 搜索历史命令 |
| `Ctrl + C` | 中断当前正在执行的命令 |
| `Ctrl + L` | 刷新屏幕 |

### 自定义别名

GDB 还允许使用 `alias` 命令自定义别名，将复杂的命令序列简化为几个字符：

```gdb
(gdb) alias bp = break
(gdb) alias w = watch
```
